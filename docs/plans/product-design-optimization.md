# MealMuse 产品设计优化设计稿

> 版本：v1.0 | 日期：2026-06-13 | 基于项目全量扫描

---

## 1. 核心定位调整

### 1.1 当前问题
产品同时做了减脂、备孕、养生、经期、家庭共享 5 个方向，每个方向浅尝辄止。

### 1.2 调整方向
- **核心路径聚焦**：饮食记录 → AI 解析 → 个性化建议，3 步是产品价值核心
- **经期模块保留差异化**：经期感知饮食是亮点，但需加深到"这周该吃什么、怎么搭配"
- **家庭共享降级**：v2 再做，v1 不投入

## 3. 饮食记录优化（P0 - 核心体验）

### 3.1 常见食物快捷录入

**新增组件：QuickFoodGrid**
```
┌─────────────────────────────────┐
│  ⚡ 快速添加                     │
│                                 │
│  [小米粥] [水煮蛋] [全麦面包]   │
│  [牛奶]   [酸奶]   [苹果]       │
│  [鸡胸肉] [西兰花] [糙米饭]     │
│  [番茄]   [黄瓜]   [玉米]       │
│                                 │
│  最近吃过 ↘                     │
│  [牛肉面] [凉拌菠菜] [豆浆]     │
│                                 │
│  ── 或者告诉 AI 你吃了什么 ──  │
│                                 │
│  💬 中午吃了一碗牛肉面...       │
│  [🎤] [📷]            [发送 📤] │
└─────────────────────────────────┘
```

**数据来源**：
- 常见食物：前端硬编码 30 个高频食物（按餐次分组）
- 最近吃过：`GET /diet/recent-foods?limit=6`，从用户最近 30 天记录中提取去重
- 点击即记录：`POST /diet/records`，用内置营养数据（不走 AI）

### 3.3 记录编辑功能
- 列表项增加"编辑"按钮，点击展开内联编辑
- 可修改：食物名称、份量、餐次

## 4. AI 对话优化（P1）

### 4.1 历史会话列表
```
┌─────────────────────────────────┐
│  💬 AI 健康助手                  │
│                                 │
│  ┌────┐                         │
│  │ 📜 │ 对话历史                 │
│  └────┘                         │
│  ├─ 最近疲劳怎么调理            │
│  ├─ 备孕期间饮食建议            │
│  ├─ 大暑节气吃什么              │
│  └─ 今日三餐推荐                │
│                                 │
│  [+ 新建对话]                   │
└─────────────────────────────────┘
```

- `GET /chat/sessions`：返回用户会话列表
- `GET /chat/sessions/{id}/messages`：加载指定会话消息
- `POST /chat/sessions`：新建会话

### 4.2 快捷问题
根据时间段动态变化：
- 早上：「今天早餐吃什么好？」「帮我安排今天三餐」
- 中午：「午餐推荐低卡高蛋白的」
- 晚上：「明天怎么安排饮食？」
- 通用：「最近营养够吗？」「经期饮食建议」

### 4.3 对话引用饮食记录
- 对话上下文自动注入用户今日饮食记录
- 用户可直接问"我今天吃了XX，怎么样？"，AI 能给出具体分析

## 5. 餐食计划优化（P1）

### 5.1 一键采纳为记录
```
┌─────────────────────────────────┐
│  🍽️ 今日餐食计划                 │
│                                 │
│  ... 三餐内容 ...               │
│                                 │
│  [✅ 采纳全部为今日记录]        │
│  [🛒 生成购物清单]              │
└─────────────────────────────────┘
```

- `POST /meals/{plan_id}/adopt`：将计划中的三餐批量创建为 diet_record
- 创建时 `source` 字段标记为 `"meal_plan"`

### 5.2 重复菜品去重
- 生成计划时传入用户近 7 天已记录的食物列表
- AI Prompt 增加"避免与最近吃过的菜重复"指令

## 7. 功能优先级矩阵

| 功能 | 优先级 | 工作量 | 用户价值 |
|------|--------|--------|---------|
| Onboarding 引导 | P0 | 3天 | 高 - 决定新用户留存 |
| 首页空状态 | P0 | 1天 | 高 - 首屏体验 |
| AI 解析 Loading | P0 | 0.5天 | 中 - 操作反馈 |
| 常见食物快捷录入 | P1 | 2天 | 高 - 降低记录成本 |
| 记录编辑 | P1 | 1天 | 中 - 基础功能补全 |
| 一键采纳计划 | P1 | 1天 | 中 - 打通计划→记录 |
| 历史对话列表 | P1 | 2天 | 中 - 对话连续性 |
| 快捷问题 | P1 | 1天 | 中 - 降低对话门槛 |
| 水摄入追踪 | P2 | 2天 | 中 - 增加日活 |
| 食物收藏 | P2 | 2天 | 低 - 便利性 |
| 周报推送 | P2 | 2天 | 中 - 增加留存 |
