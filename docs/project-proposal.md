# MealMuse 项目立项文档

> 「每一餐，都是对自己的善待」

## 1. 项目背景

### 1.1 市场痛点

| 痛点 | 描述 |
|------|------|
| **饮食规划难** | 大多数人不知道每餐该吃什么，尤其有减肥/备孕等特定目标时 |
| **记录繁琐** | 现有饮食记录 App 操作复杂，需要手动搜索食物、称重、计算热量，坚持率低 |
| **建议泛化** | 健康建议千篇一律，缺乏基于个人近期饮食的个性化指导 |
| **知识碎片化** | 养生/备孕饮食知识分散在各处，缺乏系统化、可执行的方案 |

### 1.2 市场规模

- 中国健康管理市场规模 2025 年预计超过 **4000 亿元**
- 在线健康管理用户超 **3 亿**
- 备孕/孕期营养细分市场年增长率 **20%+**
- AI+健康赛道融资活跃，用户付费意愿高

## 2. 产品愿景

**MealMuse** 是一款 AI 驱动的智能饮食管理助手，帮助用户：

- 🎯 **每日三餐智能生成** — 根据健康目标自动规划餐食
- 📝 **轻松记录饮食** — 自然语言输入，AI 自动解析营养成分
- 💬 **AI 健康对话** — 结合近期饮食数据，提供个性化养生建议
- 📊 **饮食健康看板** — 可视化营养摄入趋势

## 3. 目标用户

### 3.1 核心用户画像

| 用户类型 | 年龄 | 特征 | 核心需求 |
|----------|------|------|----------|
| **减肥人群** | 22-35 | 上班族为主，有减重目标 | 控制热量、营养均衡、简单易执行 |
| **备孕/孕期女性** | 25-38 | 关注营养摄入，有特殊饮食需求 | 叶酸/铁/钙等营养素管理、食补方案 |
| **养生爱好者** | 28-50 | 注重健康，愿意投入时间管理饮食 | 中医食疗、节气饮食、体质调理 |
| **健身人群** | 20-40 | 有增肌/塑形目标 | 蛋白质摄入管理、训练日饮食配合 |

### 3.2 用户场景

**场景一：小美（减肥）**
> 小美今天中午在公司食堂吃了一份红烧肉饭，打开 MealMuse 输入"中午吃了红烧肉+米饭+青菜"。AI 自动识别热量约 650kcal，并提醒今日热量预算还剩 800kcal，推荐晚餐吃一份鸡胸肉沙拉。

**场景二：阿丽（备孕）**
> 阿丽正在备孕，MealMuse 每天为她推荐富含叶酸和铁的餐食。她问 AI："最近总是疲劳，饮食上需要调整什么？"AI 结合她近一周的饮食记录，发现铁和 B12 摄入不足，推荐了菠菜猪肝汤等食补方案。

**场景三：老王（养生）**
> 今天是大暑节气，MealMuse 主动推送节气饮食建议："大暑宜清热祛湿，推荐冬瓜排骨汤、绿豆百合粥"。老王还可以和 AI 聊聊自己的体质，获得个性化食疗方案。

## 4. 竞品分析

| 维度 | 薄荷健康 | Keep | MyFitnessPal | **MealMuse** |
|------|----------|------|--------------|--------------|
| **核心功能** | 食物库+热量计算 | 运动+饮食 | 热量追踪 | AI 餐食生成+对话 |
| **AI 能力** | 弱（规则推荐） | 无 | 无 | **强（LLM 对话+分析）** |
| **记录体验** | 搜索选择，操作多 | 简单记录 | 搜索+扫码 | **自然语言输入** |
| **个性化程度** | 中 | 低 | 中 | **高（结合历史数据）** |
| **养生/备孕** | 弱 | 无 | 无 | **强（垂直场景）** |
| **餐食规划** | 有（模板化） | 无 | 无 | **AI 智能生成** |
| **付费意愿** | 高 | 高 | 中 | **高（健康刚需）** |

### 竞争优势

1. **AI 原生** — 不是给旧产品加 AI，而是以 AI 为核心重新设计体验
2. **自然语言交互** — 说人话就能记录、就能获得建议
3. **垂直场景深耕** — 减肥/备孕/养生，每个场景做深做透
4. **数据驱动对话** — AI 结合你的历史饮食，而不是泛泛而谈

## 5. MVP 范围定义

### 5.1 功能清单（MoSCoW 优先级）

#### Must Have（MVP 必须）
- [x] 用户注册/登录（手机号 + 微信）
- [x] 健康目标设置（减肥/备孕/养生/增肌）
- [x] 每日三餐智能生成
- [x] 饮食记录（自然语言输入）
- [x] AI 营养分析（自动识别食物热量和营养素）
- [x] AI 对话（结合近期饮食的健康问答）
- [x] 每日/每周饮食报告

#### Should Have（MVP 后快速迭代）
- [ ] 食物图片识别（拍照记录）
- [ ] 节气/时令饮食推荐
- [ ] 体质测试 & 食疗方案
- [ ] 饮食提醒通知

#### Could Have（后续版本）
- [ ] 社区分享（晒餐/打卡）
- [ ] 家庭成员管理
- [ ] 食谱收藏 & 分享
- [ ] 运动数据同步

#### Won't Have（暂不做）
- [ ] 电商/食材购买
- [ ] 在线问诊
- [ ] 运动计划

### 5.2 里程碑规划

| 阶段 | 时间 | 交付物 | 关键指标 |
|------|------|--------|----------|
