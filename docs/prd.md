# MealMuse 产品需求文档 (PRD)

> 版本：v1.1 | 更新日期：2026-06-07 | 新增：女性健康、水果推荐、家庭共享、身体数据、偏好管理

---

## 1. 产品概述

MealMuse 是一款 AI 驱动的智能饮食管理助手，通过每日三餐智能生成、自然语言饮食记录、AI 健康对话、**生理期感知饮食**、**家庭/夫妻共享**、**水果专项推荐**六大核心能力，帮助用户实现减肥、备孕、养生等健康目标。

**核心差异化：** 市面无一款 App 同时做好「生理期感知饮食 + 家庭共享 + 水果推荐 + AI 深度对话」，这是 MealMuse 的蓝海机会。

### 5.2 核心表字段说明

#### users 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| phone | VARCHAR(20) | 手机号（唯一） |
| nickname | VARCHAR(50) | 昵称 |
| height | DECIMAL(5,1) | 身高 cm |
| weight | DECIMAL(5,1) | 体重 kg |
| age | INT | 年龄 |
| gender | ENUM | 性别 |
| preferences | JSONB | 饮食偏好（素食/过敏原/口味） |
| goal_type | ENUM | health/goal 减肥/备孕/养生/增肌 |

#### diet_records 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 外键→users |
| meal_type | ENUM | breakfast/lunch/dinner/snack |
| food_text | TEXT | 用户原始输入 "中午吃了牛肉面+苹果" |
| parsed_foods | JSONB | AI 解析结果 [{name, amount, calories, ...}] |
| total_calories | INT | 总热量 kcal |
| nutrients | JSONB | 营养素汇总 {protein, fat, carbs, fiber, ...} |
| recorded_at | TIMESTAMP | 记录时间 |

#### meal_plans 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 外键→users |
| date | DATE | 日期 |
| breakfast | JSONB | 早餐方案 {name, foods[], calories, nutrients} |
| lunch | JSONB | 午餐方案 |
| dinner | JSONB | 晚餐方案 |
| total_calories | INT | 全天总热量 |
| status | ENUM | pending/completed/partial |

#### ai_chats 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 外键→users |
| session_id | UUID | 会话 ID |
| role | ENUM | user/assistant/system |
| content | TEXT | 消息内容 |
| context_snapshot | JSONB | 对话时的用户上下文快照 |
| model_used | VARCHAR(50) | 使用的模型 |
| tokens_used | INT | 消耗 token 数 |
| created_at | TIMESTAMP | 创建时间 |

## 7. 非功能需求

### 7.1 性能
- 首屏加载 < 2s
- AI 对话响应 < 3s（首 token）
- 餐食生成 < 5s

### 7.2 安全
- 用户数据加密存储
- AI 输出内容安全过滤
- 免责声明：AI 建议不替代专业医疗意见

### 7.3 合规
- 个人信息保护法合规
- 健康数据特殊保护
- AI 生成内容标识

### 7.4 可用性
- 移动端优先（响应式设计）
- 支持 PWA（后续）
- 深色模式
