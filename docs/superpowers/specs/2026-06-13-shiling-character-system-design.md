# 食灵（Shokling）角色体系设计

> 版本：v1.0 | 日期：2026-06-13 | 状态：待评审

---

## 1. 概述

### 1.1 目标

通过一个贯穿产品的角色「食灵」，为 MealMuse 建立情感连接和记忆点，让健康饮食记录从"工具行为"变成"与伙伴一起成长的旅程"。

### 1.2 角色设定

食灵来自"食材星球"，以初始形态 🥟（小笼包）降临在用户的手机里。它会根据用户的饮食记录习惯和质量，逐步进化成不同形态，用丰富的表情和性格化语言陪伴用户每一天。

### 1.3 设计原则

- **轻量实现**：视觉先用 emoji + CSS 占位，未来替换为设计稿。组件抽象良好，换皮肤不改逻辑。
- **渐进披露**：用户不会一次性看到所有功能，随着使用逐步发现。
- **正向激励**：所有的反馈都是为了鼓励健康行为，不制造焦虑。
- **本土化**：语气、食物、文化梗都贴合中文用户。

## 2. 角色外观体系（未来由设计师产出）

当前阶段使用 emoji 占位，预留好组件接口。

### 2.1 表情

**文件：** `components/ui/shokling/shokling.tsx`
**接口设计：**

```tsx
<Shokling 
  mood="happy"           // happy | celebrate | thinking | sleepy | embarrassed | fire | sad | evolve
  size="md"              // sm | md | lg
  showName?: boolean     // 是否显示名字标签
  className?: string
/>
// render 的 emoji 由 mood 映射表决定
```

**8 种核心表情：**

| mood | emoji | 触发场景 |
|------|-------|---------|
| happy | 😊 | 正常记录饮食后 |
| celebrate | 🎉 | 达成里程碑/连续记录 |
| thinking | 🤔 | AI 分析中 |
| sleepy | 😴 | 22:00 后记录 |
| embarrassed | 😅 | 记录高热量/垃圾食品 |
| fire | 🔥 | 连续打卡鼓励 |
| sad | 😢 | 超过 2 天未记录 |
| evolve | ✨ | 形态进化时 |

### 2.2 形态进化树

**文件：** `components/ui/shokling/evolution.ts`
**数据驱动：**

```ts
export interface EvolutionStage {
  id: string;
  emoji: string;
  name: string;
  description: string;
  requiredDays: number;    // 最低记录天数
  requiredScore: number;   // 最低营养评分均值
  unlockedMessage: string; // 解锁时的文案
}
```

**初始阶段（新手期）：**

| 阶段 | emoji | 名称 | 解锁条件 |
|------|-------|------|---------|
| 蛋 | 🥚 | 食灵蛋 | 注册完成 |
| 小笼包 | 🥟 | 小笼包 | 记录 3 天 |
| 嫩芽 | 🌱 | 嫩芽灵 | 记录 7 天 |

**成长阶段（按照 quality 解锁）：**

| 阶段 | emoji | 名称 | 累计记录天数 | 营养均分要求 | moving avg 窗口 |
|------|-------|------|------------|------------|----------------|
| 绿叶 | 🌿 | 绿叶灵 | ≥ 14 天 | ≥ 60 | 最近 7 天 |
| 火花 | 🔥 | 火花灵 | ≥ 30 天 | ≥ 70 | 最近 14 天 |
| 星辰 | ⭐ | 星辰灵 | ≥ 60 天 | ≥ 80 | 最近 30 天 |
| 彩虹 | 🌈 | 彩虹灵 | ≥ 100 天 | ≥ 85 | 最近 30 天 |

- "累计记录天数"：从注册日到现在的所有有记录的天数总和
- "营养均分"：最近 N 天的 daily_score 的 moving average，N 见窗口列
- 两项条件**同时满足**才解锁

```
示例：绿叶灵
累计记录天数 ≥ 14天  AND  最近7天的 daily_score 均值 ≥ 60
```

**特殊形态（隐藏成就）：**

| 阶段 | emoji | 名称 | 解锁条件 |
|------|-------|------|---------|
| 火焰烤肉 | 🥩 | 肉食者 | 连续 7 天蛋白质 > 80g/天 |
| 甜点恶魔 | 🍭 | 糖分失控 | 一天内糖分超标 3 倍（恶搞形态） |
| 养生仙人 | 🧘 | 养生仙人 | 连续 30 天 22:00 前记录 |

### 2.3 营养评分公式

```
daily_score = (protein_score × 0.3 + fiber_score × 0.2 + variety_score × 0.2 + regularity_score × 0.3) × 100

protein_score  = min(1.0, actual_protein / target_protein)
fiber_score     = min(1.0, actual_fiber / 25)
variety_score   = min(1.0, unique_food_types / 8)
regularity_score = 有三餐记录 ? 1.0 : 只有两餐 ? 0.7 : 只有一餐 ? 0.4 : 0
```

进化门槛使用**最近 N 天的 moving average**，如绿叶灵要求最近 7 天均分 ≥ 60。

### 3.2 语气风格指南

- 使用拟声词：嗯！哇！啊哈！
- 适当使用语气词：啦、呀、哦、嘛、呢
- 多用感叹句，少用句号
- 偶尔自嘲/卖萌
- 不使用否定句"你做得不对"，改为"下次可以试试..."
- 不使用医疗建议语气

## 4. App 中的核心触点

### 4.1 首页（最核心）

**位置：** 右上角或中央偏上（替换目前静态问候）

```
┌──────────────────────────────┐
│  🥟 食灵 · Lv.3              │
│  "早呀！今天状态不错哦 🌟"    │
│  ████████░░░░ 7/14 → 🌿      │
│  再坚持 7 天就能进化了!       │
├──────────────────────────────┤
│  📊 今日营养                  │
│  ...                          │
└──────────────────────────────┘
```

- 每日首次加载：根据时间段问候 + 角色表情 + 进度条
- 点击角色：弹出角色状态面板（当前形态、经验值、下一形态预览）

### 4.2 饮食记录后反馈

**位置：** 提交记录后的 toast / 浮层

原本：
```
记录成功 🎉
```

新：
```
🥟 "哇 3个牛肉包+一碗鸡蛋汤=588kcal！
     蛋白质拉满 💪 但是蔬菜少了点哦～"
```

- 提交后出现 3 秒自动消失
- 内容动态生成：分析营养构成 + 角色表情 + 一句点评
- 如果有改善空间，用"可以试试..."而不是"缺少..."

### 4.3 首页空状态（今天还没记录）

原本：
```
🍽️ 今天还没记录
试试告诉 AI 你吃了什么
```

新：
```
🥟 (委屈表情) "我等了一整天都没等到你...😢"
"现在记录还来得及！我给你准备了小惊喜哦 ✨"
[开始记录]
```

### 4.4 AI 对话

原本：纯文本对话

新：对话中食灵作为"中间人"出现

```
你好！我是你的 AI 饮食健康助手 🌿
    ↓
🥟 "嗨！是我呀～有什么想问的尽管说！
     营养师姐姐/哥哥会帮你解答的 🎤"
```

- 对话中食灵以头像形式出现在 AI 回复旁
- 发送消息时食灵有"等待中"动画

### 4.5 里程碑庆祝

- 连续记录：触发庆祝动画 + 特殊表情
- 形态进化：全屏点赞特效
- 目标达成：角色特殊反馈

## 5. 技术实现

### 5.1 目录结构

```
components/ui/shokling/
├── shokling.tsx          # 角色展示组件
├── shokling-bubble.tsx   # 角色对话框组件
├── shokling-mascot.tsx   # 首页角色卡片
├── evolution.ts          # 进化数据定义
└── index.ts              # 统一导出

lib/shokling/
├── messages.ts           # 文案模板
├── score.ts              # 营养评分计算
├── evolution-engine.ts   # 进化判定引擎
└── index.ts
```

### 5.2 数据模型（API）

```json
{
  "user_id": "uuid",
  "stage_id": "xiaolongbao",          // 当前形态
  "stage_emoji": "🥟",                // 当前 emoji
  "streak_days": 5,                   // 当前连续记录天数
  "longest_streak": 12,               // 最长连续记录
  "total_days": 23,                   // 累计记录天数
  "avg_nutrition_score": 65.3,        // 最近 7 天营养均分
  "next_stage": {                     // 下一形态
    "id": "leaf",
    "emoji": "🌿",
    "name": "绿叶灵",
    "progress": 0.7                   // 0~1
  },
  "unlocked_stages": ["egg", "xiaolongbao", "leaf"],
  "mood": "happy",
  "last_message": "今天也要加油哦！"
}
```

### 5.3 API 端点

```
GET  /api/v1/shokling/status    → 获取食灵当前状态
POST /api/v1/shokling/check-evolution → 手动触发进化检查
```

### 5.4 前端状态管理

**文件：** `stores/shokling.ts`

```ts
// Zustand store
interface ShoklingState {
  status: ShoklingStatus | null;
  loading: boolean;
  fetchStatus: () => Promise<void>;
  getMood: () => Mood;           // 根据当前场景 + 状态计算表情
  getMessage: (scene: string) => string; // 获取性格化文案
}
```

### 5.5 进化判定逻辑

1. 用户每次提交饮食记录后，前端调用 `POST /shokling/check-evolution`
2. 后端计算最近 N 天的营养评分 moving average
3. 如果满足下一形态条件 → 更新 stage → 返回 `{ evolved: true, newStage: {...} }`
4. 前端收到进化信号 → 展示庆祝动画
